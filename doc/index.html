What Is It

Sage is a run-time code analyzer for your Tcl and Tk applications, also know as a profiler. When you run your programs with Sage, you can collect and view this type of data:
  1. total execution time of program
  2. total time spent in global scope
  3. total time spent in each procedure
  4. total time spent in each procedure and its descendants
  5. number of times each procedure is called
Time data can be displayed in 3 formats:
  1. clicks (output from clock clicks)
  2. seconds
  3. a percentage of the total time
This information is useful in pinpointing performance bottlenecks in your programs, and it provides a reference for selectively optimizing your code.

To look at the user's guide, CLICK HERE.

Sage should work on any Tcl/Tk 8.0 or greater platform, although it has only been tested on Linux i386 and SunOS/Solaris.

Future Versions

Here are some features to come in future versions:
  • Windows support
  • Option to selectively wrap commands (as opposed to procs) for data collection.  This means you will be able to profile C-compiled Tcl commands as well as Tcl/Tk commands like lindex or update.
  • Create a new scope called .bind., which contains the time data for statements run as bind callbacks.
  • Speed up data collection procs via C commands.
  • Ability to wrap Tk widgets as they are created in order to gather data on widget commands.
  • Spruce up print dialog and support prettier PS output.
  • Improve interface to filters
  • Run program in its own interpreter.
  • A GUI for sage to better specify what data you want collected.
  • Statement performance breakdowns: collect data on a statement by statement basis, instead of proc by proc. Perhaps this will be selective by proc; otherwise it might be too slow to do statement analysis for entire program. For instance, user can select to do statement analysis for the procs x, y, and z.
  • Proc details. For each proc, keep track of the following data:
    • who called me and how many times
    • who do I call, how many times, and how much time do I spend in each descendant
  • Call graph? Is this sort of thing useful?
  • Test coverage support?
